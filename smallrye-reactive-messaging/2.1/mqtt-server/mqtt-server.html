MQTT Server

The MQTT Server connector starts an MQTT server and forwards messages sent from MQTT clients to your Reactive Messaging application. The MQTT Server connector is based on the Vert.x MQTT Server.

Introduction

MQTT is a machine-to-machine (M2M)/"Internet of Things" connectivity protocol. It was designed as an extremely lightweight publish/subscribe messaging transport.

Unlike the MQTT Connector, this connector starts an MQTT server and receives messages from connected clients. These messages are routed to your Reactive Messaging application.

This isn’t a fully featured MQTT server, it will only handle publish requests and their acknowledgment (for QoS 1 & 2). It will ignore subscription requests for instance.
The server has no persistence, if no subscribers are listening, messages will be dropped.

Using the MQTT Server connector

To you the MQTT Server Connector, add the following dependency to your project:

<dependency>
  <groupId>io.smallrye.reactive</groupId>
  <artifactId>smallrye-reactive-messaging-mqtt-server</artifactId>
  <version>2.1.0</version>
</dependency>

The connector name is: smallrye-mqtt-server.

So, to indicate that a channel is managed by this connector you need:

# Inbound
mp.messaging.incoming.[channel-name].connector=smallrye-mqtt
The MQTT Server connector only implement the inbound connector facet.

Receiving messages from MQTT Clients

The MQTT Server Connector starts an MQTT server and dispatches the messages sent by connected clients. It maps each of them into Reactive Messaging Messages.

Example

Configure your application to receive MQTT messages on the prices channel as follows:

mp.messaging.incoming.prices.connector=smallrye-mqtt-server (1)

  1. Sets the connector for the prices channel

By default, the server starts on the host/interface 0.0.0.0 and port 1883. This can be overridden with the host and port attributes.
Message coming from MQTT have a byte[] payload.

Then, your application receives Message<byte[]>. You can consumes the payload directly:

package inbound;

import org.eclipse.microprofile.reactive.messaging.Incoming;

import javax.enterprise.context.ApplicationScoped;

@ApplicationScoped
public class MqttServerPriceConsumer {

    @Incoming("prices")
    public void consume(byte[] raw) {
        double price = Double.parseDouble(new String(raw));

        // process your price.
    }

}

Or, you can retrieve the Message<byte[]>:

package inbound;

import org.eclipse.microprofile.reactive.messaging.Incoming;
import org.eclipse.microprofile.reactive.messaging.Message;

import javax.enterprise.context.ApplicationScoped;
import java.util.concurrent.CompletionStage;

@ApplicationScoped
public class MqttServerPriceMessageConsumer {

    @Incoming("prices")
    public CompletionStage<Void> consume(Message<byte[]> price) {
        // process your price.

        // Acknowledge the incoming message
        return price.ack();
    }

}

Deserialization

The MQTT Connector does not handle the deserialization and creates a Message<byte[]>.

Inbound Metadata

The MQTT connector does not provide inbound metadata.
