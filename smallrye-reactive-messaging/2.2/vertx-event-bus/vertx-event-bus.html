Vert.x Event Bus

The Vert.x Event Bus connector adds support for the Vert.x Event Bus to Reactive Messaging.

It allows a Reactive Messaging application to send and receive messages from the Vert.x event bus. It is mainly designed to ease the interaction with Vert.x applications.

Introduction

The Vert.x event bus connector is able to:

  • receive messages from the Vert.x event bus and send them to the associated channel

  • send messages to the Vert.x event bus for each Message transiting on a channel

Using the Vert.x Event Bus connector

To you the Vert.x Event Bus Connector, add the following dependency to your project:

<dependency>
  <groupId>io.smallrye.reactive</groupId>
  <artifactId>smallrye-reactive-messaging-vertx-eventbus</artifactId>
  <version>2.2.1</version>
</dependency>

The connector name is: smallrye-vertx-eventbus.

So, to indicate that a channel is managed by this connector you need:

# Inbound
mp.messaging.incoming.[channel-name].connector=smallrye-vertx-eventbus

# Outbound
mp.messaging.outgoing.[channel-name].connector=smallrye-vertx-eventbus

Receiving messages from the Vert.x Event Bus

The Vert.x Event Bus Connector retrieves Event Bus Message and maps each of them into Reactive Messaging Messages. These messages are sent to the associated channel.

Example

Configure your application to receive Event Bus Message on the prices channel as follows:

mp.messaging.incoming.prices.connector=smallrye-vertx-eventbus       (1)
mp.messaging.incoming.prices.address=the-eventbus-address            (2)

  1. Sets the connector for the prices channel

  2. Sets the address attribute - it’s the listened event bus address

Then, your application receives Message<Double>. You can consumes the payload directly:

package inbound;

import org.eclipse.microprofile.reactive.messaging.Incoming;

import javax.enterprise.context.ApplicationScoped;

@ApplicationScoped
public class EventBusPriceConsumer {

    @Incoming("prices")
    public void consume(double price) {
        // process your price.
    }

}

Or, you can retrieve the Message<Double>:

package inbound;

import org.eclipse.microprofile.reactive.messaging.Incoming;
import org.eclipse.microprofile.reactive.messaging.Message;

import javax.enterprise.context.ApplicationScoped;
import java.util.concurrent.CompletionStage;

@ApplicationScoped
public class EventBusPriceMessageConsumer {

    @Incoming("prices")
    public CompletionStage<Void> consume(Message<Double> price) {
        // process your price.

        // TASK - Explain acknowledgement
        // Acknowledge the incoming message
        return price.ack();
    }

}

Deserialization

The deserialization is handled by the Vert.x Event Bus. You need to configure the event bus sender to receive the right payload type.

Acknowledgement

The Vert.x event bus does not support acknowledgement. However it supports replies. If the use-reply-as-ack attribute is set to true, the connector sends a reply to the Event Bus Message when the produced Reactive Messaging message is acknowledged. The reply sent a simple "OK".

Sending Vert.x Event Bus messages

The Vert.x Event Bus Connector can send/publish Reactive Messaging Messages as Event Bus Message.

Example

Configure your application to send the messages from the prices channel on the event bus as follows:

mp.messaging.outgoing.prices.connector=smallrye-vertx-eventbus       (1)
mp.messaging.outgoing.prices.address=<the-eventbus-address>          (2)

  1. Sets the connector for the prices channel

  2. Sets the event bus address on which the messages are sent

Then, your application must send Message<Double> to the prices channel. It can use double payloads as in the following snippet:

package outbound;

import io.smallrye.mutiny.Multi;
import org.eclipse.microprofile.reactive.messaging.Outgoing;

import javax.enterprise.context.ApplicationScoped;
import java.time.Duration;
import java.util.Random;

@ApplicationScoped
public class EventBusPriceProducer {

    private Random random = new Random();

    @Outgoing("prices")
    public Multi<Double> generate() {
        // Build an infinite stream of random prices
        // It emits a price every second
        return Multi.createFrom().ticks().every(Duration.ofSeconds(1))
            .map(x -> random.nextDouble());
    }

}

Or, you can send Message<Double>:

package outbound;

import io.smallrye.mutiny.Multi;
import org.eclipse.microprofile.reactive.messaging.Message;
import org.eclipse.microprofile.reactive.messaging.Outgoing;

import javax.enterprise.context.ApplicationScoped;
import java.time.Duration;
import java.util.Random;

@ApplicationScoped
public class EventBusPriceMessageProducer {

    private Random random = new Random();

    @Outgoing("prices")
    public Multi<Message<Double>> generate() {
        // Build an infinite stream of random prices
        // It emits a price every second
        return Multi.createFrom().ticks().every(Duration.ofSeconds(1))
            .map(x -> Message.of(random.nextDouble()));
    }

}

Serialization

The serialization is handled by teh Vert.x Event Bus. You can configure the codec attribute to indicate which codec should be used to encode the payload:

mp.messaging.outgoing.prices.connector=smallrye-vertx-eventbus
mp.messaging.outgoing.prices.address=<the-eventbus-address>
mp.messaging.outgoing.prices.codec=PriceCodec

The codec must have been registered on the event bus beforehand.

Acknowledgement

The Vert.x event bus does not support acknowledgement. However it provides reply support.

You can set the expect-reply attribute to indicate that the Reactive Messaging message must be acknowledged when the reply is received. The timeout attribute lets you configure how long this reply is awaited.

Note that expect-reply is incompatible with the publish attribute. If expect-reply is not set, the message is acknowledged just after having been sent/published on the event bus.
