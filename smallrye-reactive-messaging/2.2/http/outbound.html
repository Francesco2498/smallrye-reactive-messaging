Sending HTTP requests

The HTTP Connector can map Reactive Messaging Messages as HTTP Request and send them to a remote HTTP service.

Example

Let’s imagine you have a HTTP service running, and accessible using the http://localhost:8089. Configure your application to write the messages from the service channel into a HTTP request sent to this service as follows:

mp.messaging.outgoing.service.connector=smallrye-http   (1)
mp.messaging.outgoing.service.url=http://localhost:8089  (2)

  1. Sets the connector for the service channel

  2. Configure the location of the HTTP service

Then, your application must send Message<String> to the service channel. It can use String payloads as in the following snippet:

package outbound;

import io.smallrye.mutiny.Multi;
import org.eclipse.microprofile.reactive.messaging.Outgoing;

import javax.enterprise.context.ApplicationScoped;
import java.time.Duration;
import java.util.Random;

@ApplicationScoped
public class HttpPriceProducer {

    private Random random = new Random();

    @Outgoing("service")
    public Multi<String> generate() {
        // Build an infinite stream of random prices
        // It emits a price every second
        return Multi.createFrom().ticks().every(Duration.ofSeconds(1))
            .map(x -> Double.toString(random.nextDouble()));
    }

}

Or, you can send Message<String>:

package outbound;

import io.smallrye.mutiny.Multi;
import org.eclipse.microprofile.reactive.messaging.Message;
import org.eclipse.microprofile.reactive.messaging.Outgoing;

import javax.enterprise.context.ApplicationScoped;
import java.time.Duration;
import java.util.Random;

@ApplicationScoped
public class HttpPriceMessageProducer {

    private Random random = new Random();

    @Outgoing("service")
    public Multi<Message<String>> generate() {
        // Build an infinite stream of random prices
        // It emits a price every second
        return Multi.createFrom().ticks().every(Duration.ofSeconds(1))
            .map(x -> Message.of(Double.toString(random.nextDouble())));
    }

}

Serialization

The HTTP connector support the following payloads:

  • String

  • io.vertx.core.buffer.Buffer

  • byte[]

  • ByteBuffer

  • JsonObject

  • JsonArray

  • Cloud Event

The outgoing HTTP request body is the serialized payload (using one of the above types). The method is either POST (default) or PUT.

Acknowledgement

The Message is acknowledged when the HTTP service sends a 2xx response.

The HTTP connector is based on the Vert.x Web Client. You can pass any supported property as channel attribute.
